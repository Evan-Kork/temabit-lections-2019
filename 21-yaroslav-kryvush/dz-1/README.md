git init - створити новий репозиторій
git clone url - скопіювати репозиторій в локальну папку з віддаленого репозиторію 
git status - перегляд, що знаходиться в робочій директорії і staging area 
git add <ім'я/файла> - добавити раніші зміни в staging area 
git checkout - перейти на гілку яка є в репозиторії
git checkout b <ім'я/гілки> = git branch <ім'я/гілки> - створити гілку 
git push - залити на сервер
git fetch - показує що в директорії є зміни
git commit - коміт в свій локальний репозиторій
git pull - отримати з сервера
git flow - це набір розширень git надає високорівневі операції над репозиторієм для підтримки моделі розгалуження Vincent Driessen
git merge - поєднання зміни із даної гілки в гілку master
